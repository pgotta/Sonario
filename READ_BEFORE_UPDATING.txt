========================================================================
  UPDATING SONARIO — PRESERVE LOCAL CREDENTIALS AND RUNTIME DATA
========================================================================

A Sonario release never includes real Google credentials, OAuth tokens, API
keys, usage state, analysis history, cache, output, logs, PID files, or the
private browser profile.

Before replacing an existing installation, back up the entire credentials\
folder, especially:

  credentials\credentials.json
  credentials\token.json
  credentials\api_keys.json

Recommended update:

  1. Close Sonario with X and confirm the window is gone.
  2. Copy credentials\ somewhere safe.
  3. Extract the new complete release over the old installation, or into a new
     folder.
  4. Restore credentials\ if using a new folder.
  5. Run setup_all.bat if dependencies or launchers changed.

The current release uses Groq Qwen 3.6 27B, recovers malformed structured model
responses instead of dropping documents, opens in a dedicated private browser
window, and stops its exact Flask backend when that window is closed with X.

BAT files are included in Windows release ZIPs but intentionally excluded from
Git. Their authoritative source is BUILD.md.
========================================================================
